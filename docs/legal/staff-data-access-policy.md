# Регламент доступа сотрудников и ролей платформы к данным

**Версия:** 1.0  
**Дата:** 01.03.2026  
**Статус:** draft

---

## 1. Цель документа

1.1. Настоящий Регламент определяет:
- какие данные доступны каждой роли на Платформе AI Strategic Academy;
- принципы разграничения доступа к данным;
- запрещённые действия с данными;
- ответственность за нарушение правил доступа.

1.2. Цель Регламента — предотвратить несанкционированный доступ к данным, защитить конфиденциальность пользователей и обеспечить соответствие принципу минимально необходимого доступа.

---

## 2. Принципы доступа

2.1. **Минимально необходимый доступ (least privilege):** каждая роль имеет доступ только к тем данным, которые необходимы для выполнения её функций.

2.2. **Role-Based Access Control (RBAC):** доступ определяется ролью пользователя в системе.

2.3. **Need-to-know:** доступ к данным предоставляется только в объёме, необходимом для выполнения конкретной задачи.

2.4. **Auditability:** все действия с данными, выходящие за рамки обычного использования, логируются.

2.5. **Запрет передачи аккаунта:** сотрудник не имеет права передавать свой аккаунт или учётные данные другому лицу.

## 3. Матрица доступа к данным

### 3.1. Регистрационные данные (имя, email, телефон, роль)

| Роль | Что видит | Что может менять | Запрещено |
|---|---|---|---|
| **Admin** | Все регистрационные данные | Имя, email, роль, статус аккаунта | Изменять данные без основания |
| **Super Curator** | Имя, роль, принадлежность к потоку | Не может менять | Просматривать email и телефон без необходимости |
| **Curator** | Имя назначенных слушателей | Не может менять | Просматривать email и телефон без необходимости |
| **Instructor** | Имя слушателей на своих курсах | Не может менять | Просматривать email и телефон без необходимости |
| **Customer Observer** | Имя (в рамках разрешённого scope) | Не может менять | Несанкционированный просмотр |
| **Student** | Только свои данные | Только свои настройки (пароль, уведомления) | Просмотр чужих данных |

### 3.2. Учебные данные (прогресс, тесты, задания)

| Роль | Что видит | Что может менять | Запрещено |
|---|---|---|---|
| **Admin** | Все учебные данные | Статус, enrollment, сертификаты | Изменять оценки и ответы без основания |
| **Super Curator** | Прогресс, риски по потокам | Не может менять | Просмотр данных вне своего scope |
| **Curator** | Прогресс, задания, тесты назначенных слушателей | Оценки, обратная связь по заданиям | Изменять ответы слушателя |
| **Instructor** | Прогресс по своим курсам | Материалы курсов, тесты, задания | Изменять ответы слушателей |
| **Customer Observer** | Агрегированные отчёты, прогресс scope | Не может менять | Просмотр индивидуальных ответов и оценок |
| **Student** | Только свои данные | Свои ответы, загрузка файлов | Просмотр чужих работ |

### 3.3. Файлы (задания, материалы, сертификаты)

| Роль | Доступ к файлам |
|---|---|
| **Admin** | Все файлы (в рамках администрирования) |
| **Curator** | Файлы заданий назначенных слушателей |
| **Instructor** | Файлы материалов своих курсов |
| **Student** | Свои файлы, файлы материалов уроков |
| **Customer Observer** | Не имеет доступа к файлам |

### 3.4. Коммуникации (чат, вопросы)

| Роль | Что видит |
|---|---|
| **Admin** | Все коммуникации (при необходимости аудита) |
| **Curator** | Только диалоги с назначенными слушателями |
| **Instructor** | Только переадресованные вопросы |
| **Student** | Только свои диалоги |
| **Customer Observer** | Не видит коммуникации |

## 4. Запрещённые действия

Сотрудникам и ролям Платформы **запрещено**:

4.1. Просматривать данные пользователей без рабочей необходимости.

4.2. Скачивать файлы и данные вне процесса выполнения служебных задач.

4.3. Передавать данные пользователей третьим лицам без законного основания или письменного разрешения.

4.4. Использовать данные пользователей в личных целях.

4.5. Делать скриншоты или копировать данные пользователей, за исключением случаев, когда это необходимо для выполнения служебных обязанностей.

4.6. Выгружать отчёты, содержащие персональные данные, без разрешения ответственного лица.

4.7. Использовать личные мессенджеры, облачные хранилища или иные неавторизованные каналы для передачи данных пользователей.

## 5. Audit logs

5.1. Следующие действия подлежат обязательному логированию:
- вход в аккаунт;
- изменение данных пользователя;
- выдача/отзыв сертификата;
- изменение прав доступа;
- изменение статуса аккаунта;
- массовые операции с данными;
- экспорт отчётов;
- попытки несанкционированного доступа.

5.2. Audit logs хранятся в течение срока, установленного Политикой хранения и удаления данных.

5.3. Просмотр audit logs доступен:
- администраторам;
- ответственному лицу за безопасность.

5.4. Audit logs не могут быть изменены или удалены рядовыми сотрудниками.

---

## 6. Нарушения и ответственность

6.1. Нарушение настоящего Регламента влечёт:
- ограничение доступа к Платформе;
- дисциплинарные меры в соответствии с трудовым договором;
- отстранение от работы с данными;
- уведомление руководства Академии.

6.2. В случае обнаружения нарушения сотрудник обязан немедленно сообщить ответственному лицу.

6.3. Факт нарушения фиксируется в audit logs и может быть передан правоохранительным органам в случаях, предусмотренных законодательством.

---

## 7. Пересмотр доступа

7.1. Доступ сотрудников к данным пересматривается:
- при изменении должностных обязанностей;
- при переводе на другую роль;
- при увольнении сотрудника;
- не реже одного раза в год.

7.2. При увольнении сотрудника его доступ к Платформе блокируется немедленно.
