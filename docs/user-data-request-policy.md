## 1. Какие запросы принимаются

Пользователь Платформы AI Strategic Academy имеет право обратиться с любым из следующих запросов:

| Тип запроса | Описание |
|---|---|
| **Исправить данные** | Уточнить или изменить неточные персональные данные (имя, email, телефон) |
| **Получить информацию** | Запросить информацию о том, какие данные обрабатываются, с какой целью и кому передаются |
| **Удалить аккаунт** | Запросить удаление аккаунта и связанных с ним данных |
| **Удалить файл** | Запросить удаление конкретного загруженного файла |
| **Уточнить сертификат** | Запросить информацию о сертификате, его статусе или дубликат |
| **Ограничить использование** | Ограничить обработку данных в определённых целях |
| **Пожаловаться на доступ** | Сообщить о неправомерном доступе к данным со стороны другого пользователя или сотрудника |
| **Отозвать согласие** | Отозвать ранее данное согласие на обработку данных (если обработка основана на согласии) |

## 3. Идентификация заявителя

3.1. Для обработки запроса Оператор должен удостовериться, что запрос исходит от самого Пользователя или его законного представителя.

3.2. Идентификация производится:
- по email, привязанному к аккаунту;
- путём подтверждения доступа к аккаунту;
- с помощью дополнительных сведений (если требуется).

3.3. Оператор вправе запросить дополнительную информацию, необходимую для идентификации, но не вправе требовать данные, не относящиеся к проверке личности.

3.4. **Оператор не передаёт данные заявителю, если личность не подтверждена.**

## 5. Процесс обработки запроса

### Шаг 1. Получение запроса
- Запрос регистрируется в журнале запросов.

### Шаг 2. Подтверждение получения
- Пользователю направляется уведомление о получении запроса с указанием даты и регистрационного номера.

### Шаг 3. Проверка личности
- Проверяется, что заявитель является владельцем аккаунта или его законным представителем.

### Шаг 4. Определение типа запроса
- Запрос классифицируется по типу (см. раздел 1).

### Шаг 5. Проверка законных и договорных ограничений
- Проверяется, не препятствуют ли законодательство, договорные обязательства или технические ограничения выполнению запроса.

### Шаг 6. Выполнение действия
- Выполняется действие, соответствующее типу запроса (исправление, удаление, предоставление информации и т.д.).

### Шаг 7. Фиксация результата
- Результат обработки запроса фиксируется в журнале.

### Шаг 8. Ответ пользователю
- Пользователь информируется о результате обработки запроса с указанием: что было сделано, какие данные сохранены (если не все удалены) и по какому основанию.

## 7. Журнал запросов

Каждый запрос регистрируется в журнале со следующими полями:

| Поле | Описание |
|---|---|
| requestId | Уникальный номер запроса |
| userId | Идентификатор пользователя |
| type | Тип запроса (исправление, удаление, информация и т.д.) |
| receivedAt | Дата и время получения |
| status | Статус обработки (new, in_progress, completed, rejected) |
| resolvedAt | Дата и время завершения обработки |
| responsiblePerson | ФИО ответственного сотрудника |
| result | Результат обработки (что сделано, основание, если отказано) |
